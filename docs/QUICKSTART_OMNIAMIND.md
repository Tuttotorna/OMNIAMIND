# Quickstart OMNIAMIND

This document gives the fastest path to seeing OMNIAMIND as a structural cognition orchestration layer.

The public mental model is:

    observation -> structural reasoning workflow -> OMNIA measurement -> limit boundary -> external decision

---

## Clean check

    git clone https://github.com/Tuttotorna/OMNIAMIND.git
    cd OMNIAMIND
    python -m pytest

If no tests are present, pytest may return exit code 5.

In that case, absence of tests is treated as a repository status signal, not as a failed implementation claim.

The documentation and manifest sanity checks remain mandatory.

---

## What to look for

The point is not to prove artificial consciousness.

The point is to identify:

    What observation enters the workflow?
    How is the workflow organized?
    Which layer receives the case?
    Where does measurement happen?
    Where does the limit boundary appear?
    Where does the workflow stop?
    What remains external?

---

## Expected orchestration behavior

A good OMNIAMIND path should produce a workflow artifact or clearly traceable routing logic.

It should not silently become:

    autonomous consciousness claim
    semantic judgment
    final decision
    truth certificate
    model intelligence claim

---

## Public compression

    OMNIAMIND organizes structural cognition workflows.
    OMNIA measures.
    Decision remains external.
